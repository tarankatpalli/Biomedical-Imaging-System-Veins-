> **This guide sets clear expectations for everyone involved. Following these guidelines ensures a positive experience for contributors and maintainers alike.** ✨

## 📖 Code of Conduct
Please review our [Code of Conduct](https://github.com/tarankatpalli/Biomedical-Imaging-System-Veins-?tab=coc-ov-file#). It is in effect at all times. Harassment or unprofessional behavior will not be tolerated. 🚫

## 💡 Asking Questions
For general questions about NIR imaging or Edge AI, use [GitHub Discussions](https://github.com). Keep **Issues** strictly for bugs and features. 💭

## 📥 Opening an Issue
Before [creating an issue](https://github.com), ensure you are on the latest version. 🚀

### 🪲 Bug Reports
*   **Search First:** Check if the bug was already reported.
*   **Hardware Context:** You **must** specify your NIR sensor, camera specs, and Edge AI device (e.g., Jetson Nano). 📸
*   **Format:** Use Markdown backticks (```) for code blocks and logs.

## 💌 Feature Requests
We love new ideas for vascular visualization! 
*   **Precision:** Explain exactly how it improves hemodynamic analysis.
*   **Details:** Provide implementation ideas for AI models or 3D reconstruction.

## 🔍 Triaging Issues
Help us by reproducing reported bugs or asking for missing info. Your help is appreciated! 🧐

## 🔁 Submitting Pull Requests
For big changes, open an issue to discuss your approach first. 🎉

*   **⚡ Efficiency:** Optimize for **Local Edge AI** (low-latency).
*   **🤏 Smaller is Better:** One PR per bug fix or feature.
*   **🐍 Style:** Follow PEP 8 for Python.
*   **🧪 Tests:** Include unit tests for new image processing filters.
*   **📖 Docs:** Update the README if you change the core pipeline.

## 📝 Writing Commit Messages
Use the **imperative mood** and be concise:
1.  `Add NIR denoising filter` (Correct)
2.  `Fixed some stuff` (Incorrect)

## ✅ Code Review
All submissions require review via GitHub Pull Requests. ✅

## 💅 Coding Style
*   **Python:** PEP 8.
*   **C++:** Focus on memory efficiency.
*   **Tabs vs Spaces:** Use **spaces** only. ⌨️

## 🏅 Certificate of Origin
By contributing, you certify you have the right to submit code under the project's license. 🥇
